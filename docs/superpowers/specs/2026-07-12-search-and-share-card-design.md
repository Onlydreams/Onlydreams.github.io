# 搜索增强与默认分享卡片设计

## 目标

在不增加服务端、搜索服务或运行时依赖的前提下，让静态搜索可以被链接直达、按文章元数据收敛，并让站内文章具备稳定的社交分享预览图。

## 范围

本次包含：

- 搜索页支持 `?q=` 查询深链接，并同步查询、分类、标签和文章状态到 URL。
- 将分类、标签和状态作为可单独使用、可组合的筛选条件；没有关键词时也能用于浏览文章。
- 标题、元数据和摘要中的关键词使用安全的 DOM 文本节点高亮，不把查询字符串拼进 HTML。
- 将搜索索引正文上限从 300 个字符扩展到 800 个字符，并带上文章状态标签。
- 生成一张 1200×630 的默认 PNG 分享卡片，并以 Jekyll front-matter defaults 应用于所有文章；文章 front matter 的 `image` 可覆盖默认卡片。
- 在功能测试中覆盖搜索页面约定、索引字段、URL 状态逻辑、默认 OG/Twitter 元数据和卡片文件。

本次不包含：

- 模糊匹配、分词、拼音搜索、搜索排序服务或第三方搜索平台。
- 对已有文章逐篇制作专属卡片。
- 任何追踪、点击统计或服务端 API。

## 搜索交互

搜索保持一个主输入框和紧凑的筛选行：分类、标签、状态均为单选下拉框，选项由静态 `search.json` 在浏览器端生成。页面加载时读取 `q`、`category`、`tag`、`status`；输入或筛选变化时使用 `history.replaceState` 更新 URL，不产生浏览历史噪声。

关键词继续采用不区分大小写的子串匹配；筛选是精确元数据匹配。结果最多展示 10 篇，标题、分类、标签、状态和摘要中会标出完整查询词。高亮必须通过 `textContent` / `createTextNode` 构造，避免 URL 参数或文章内容导致注入。

视觉上延续站点的暖灰编辑风格：输入框为主，筛选栏用细边框和小字号收束，结果元数据为低对比文本，高亮只用柔和橙色底色，不使用卡片堆叠或动画噪声。

## 默认分享卡片

卡片尺寸为 1200×630，使用站点暖灰、炭黑和橙色强调色，表达“技术笔记 / 工具链 / 排障”的编辑式索引感。它是文章的兜底图，不承担每篇文章的具体主题；重点文章只需在 front matter 添加 `image` 即可覆盖。

`_config.yml` 的 `defaults` 只对 `type: posts` 写入下列默认值：

```yaml
image:
  path: /assets/images/onlydreams-og-card.png
  width: 1200
  height: 630
  alt: Onlydreams 技术笔记
```

这会由已启用的 `jekyll-seo-tag` 输出 `og:image`、`twitter:image` 及图片尺寸和替代文本。

## 验收

- `/search/?q=codex` 打开后自动运行查询，筛选变化后 URL 保持可复制。
- 分类、标签、状态可独立或组合筛选；关键词命中显示高亮，未命中显示清晰空状态。
- `search.json` 每项包含状态，正文长度不超过 800 个字符。
- 至少一篇构建后的文章输出默认 `og:image` 与 `twitter:image`，指向绝对的默认 PNG URL。
- `assets/images/onlydreams-og-card.png` 存在，尺寸为 1200×630。
- `git diff --check` 与 `./bin/test.ps1` 通过。
